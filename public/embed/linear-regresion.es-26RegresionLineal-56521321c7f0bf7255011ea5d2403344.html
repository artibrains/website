# Bloque de C√≥digo 2: Creando un conjunto de datos sint√©tico y realista

# Estoy configurando una 'semilla aleatoria' aqu√≠. Este es un paso crucial en la ingenier√≠a de software
# para el aprendizaje autom√°tico. Asegura que nuestros n√∫meros 'aleatorios' sean los mismos cada

np.random.seed(42)

# Simulemos datos para 50 d√≠as.

Paso 3: El Cerebro de la Operaci√≥n (Los Componentes del Modelo)¬∂

Sophia nos dio el plano, ahora definamos las piezas centrales en nuestro c√≥digo:

  • La Hip√≥tesis ($h(x)$): Esta es nuestra funci√≥n de predicci√≥n. Para la regresi√≥n lineal, es solo la ecuaci√≥n de una l√≠nea. La llamar√© prediction(). $$prediction(x) = w * x + b$$ $w$ (peso) y $b$ (sesgo) son los par√°metros que nuestro modelo aprender√°. Estos son los "mandos" que ajustaremos.
  • La Funci√≥n de Costo ($J(w, b$)): Esto nos dice cu√°n "equivocadas" est√°n las predicciones de nuestro modelo. Cuanto menor, mejor. Usaremos el Error Cuadr√°tico Medio (MSE), una elecci√≥n bastante est√°ndar. $$cost = averageOf( (prediction - actualvalue)^2 )$$
  • Descenso de Gradiente: Este es el proceso de aprendizaje en s√≠. Ajusta autom√°ticamente $w$ y $b$ para minimizar el costo. Descubre la "pendiente" de la funci√≥n de costo y da pasos cuesta abajo.

Paso 4: C√≥mo Aprende el C√≥digo (El Bucle de Entrenamiento)¬∂

Este es el coraz√≥n de nuestro programa, la sala de m√°quinas. Inicializaremos nuestros par√°metros y luego ejecutaremos un bucle que los refinar√° repetidamente utilizando el Descenso de Gradiente. Comentar√© cada l√≠nea para que puedas ver exactamente lo que est√° sucediendo.

Al tratarse de una muestra de datos muy peque√±a con solo 50 elementos y para mantener el codigo senzillo, utilizaremos el descenso por lotes utilizando las 50 entradas en cada iteraci√≥n.

In¬†[¬†]:
# Bloque de C√≥digo 3: Construyendo y Entrenando el Modelo (con Detenci√≥n Temprana)

# --- 1. Inicializaci√≥n de Par√°metros ---
# Necesitamos darle a nuestro modelo un punto de partida. Empecemos w y b en 0.
# Es una posici√≥n inicial simple y neutral.
w = 0.0
b = 0.0

# --- 2. Configuraci√≥n de Hiperpar√°metros ---
# Estos son ajustes para el algoritmo de aprendizaje, no aprendidos de los datos.
# Tenemos que configurarlos nosotros mismos.
learning_rate = 0.0001     # Qu√© tan grande es el paso a dar en cada actualizaci√≥n. Un valor peque√±o es m√°s seguro.
num_iterations = 10000      # El n√∫mero m√°ximo de iteraciones.
alpha = 0.001              # Cambio m√≠nimo en el costo para seguir adelante (umbral de detenci√≥n temprana).

# --- 3. El Bucle de Entrenamiento ---
# Almacenaremos el costo en cada paso para observar c√≥mo aprende nuestro modelo.
cost_log = []

print("Iniciando la secuencia de entrenamiento...")
for i in range(num_iterations):
    # a) Realizar predicciones para TODOS nuestros puntos de datos con los w y b actuales.
    y_pred = w * X + b

    # b) Calcular el costo (MSE) para estas predicciones.
    # Esto nos dice qu√© tan bien lo estamos haciendo en esta iteraci√≥n.
    cost = np.mean((y_pred - y)**2)
    cost_log.append(cost)

    # --- Condici√≥n de Detenci√≥n Temprana ---
    # Si el costo no ha cambiado mucho en comparaci√≥n con el paso anterior,
    # significa que hemos alcanzado una regi√≥n plana, no estamos aprendiendo mucho m√°s.
    if i > 0 and abs(cost_log[-2] - cost_log[-1]) < alpha:
        print(f"Deteniendo temprano en la iteraci√≥n {i} debido a un peque√±o cambio en el costo.")
        break

    # c) Calcular los gradientes. Este es el n√∫cleo del aprendizaje.
    # Estas f√≥rmulas provienen de la derivada de c√°lculo de la funci√≥n de costo.
    # Nos dicen la pendiente de nuestra "colina" de costo para w y b.
    dw = (1/m) * np.sum((y_pred - y) * X)  # Gradiente para el peso
    db = (1/m) * np.sum(y_pred - y)        # Gradiente para el sesgo

    # d) Actualizar los par√°metros.
    # Movemos w y b en la direcci√≥n *opuesta* a su gradiente.
    # Este es el paso "cuesta abajo".
    w = w - learning_rate * dw
    b = b - learning_rate * db

    # Un registro simple para monitorear el proceso en tiempo real.
    # Imprimimos la evoluci√≥n del error cada 100 iteraciones
    # Empezamos imprimiendo la primera iteraci√≥n ya que el error en 0 no es significativo al ser w y b aleatorios
    if ((i-1) % 100) == 0:
        print(f"Iteraci√≥n {i}: Costo = {cost:.6f}")

# imprimir el n√∫mero de iteraciones y el error final
print("\n--- Entrenamiento Completo ---")
print(f"N√∫mero de iteraciones: {i}")
print(f"Error final: {cost:.6f}")
print(f"Peso aprendido final (w): {w:.4f}")
print(f"Sesgo aprendido final (b): {b:.4f}")
